# Platform reference

Rough norms only; platforms change UI and limits over time. Prefer the user's stated cap when they give one.

| Platform | Typical short length | Notes |
|----------|----------------------|--------|
| X | ~280 chars (longer with subscription tiers) | Line breaks OK; front-load the hook |
| LinkedIn | ~1–3 short paragraphs | White space helps; first line is the hook in feed |
| Bluesky | ~300 chars default; longer posts available | Conversational; avoid corporate cadence unless asked |
| Mastodon | 500 chars default (instance may vary) | Content warnings when discussing sensitive topics; fewer hashtags by default |
| Blog / newsletter | 400–900 words default in skill | Subheads optional; link when user provides URL |

## Hashtags and mentions

- Default: **zero to two** hashtags unless the user wants discovery-heavy copy.
- Do not @-mention people or brands unless the user asks.

## Links and CTAs

- Use placeholders like `[link]` when no URL is given.
- One primary CTA per piece unless the user asks for more.
